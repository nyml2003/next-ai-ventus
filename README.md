# next-ai-ventus

> 面向开发者的轻量级博客系统 —— 用管理代码的方式管理内容

---

## 这是什么

一个为技术博主设计的博客系统。

不同于 WordPress 的臃肿、Notion/语雀的封闭、或静态博客的局限，ventus 提供：

- **内容的完全掌控** - 文章是本地 Markdown 文件，用 Git 管理版本
- **部署的极致简单** - 单二进制文件 + 文件目录，2核2G 服务器即可运行
- **体验的现代化** - React SPA 级的交互，但保留 MPA 的简洁和 SEO 友好
- **开发的自由度** - 前后端分离，API 清晰，易于二次开发

---

## 为谁而建

| 用户 | 痛点 | ventus 的解决方案 |
|------|------|------------------|
| **技术博主** | 想专注写作，不想维护复杂系统 | 文件即内容，Git 提交即发布 |
| **开发者** | 现有博客系统太封闭，无法定制 | 开源代码，自定义主题，扩展模块 |
| **独立站长** | 服务器资源有限，怕流量大了扛不住 | 无数据库，内存占用 < 20MB，静态资源走 CDN |
| **内容创作者** | 担心平台封禁、数据丢失 | 内容完全本地存储，可导出、可迁移 |

## 核心特性

### 📝 写作体验

- **本地 Markdown 编辑** - 用你喜欢的编辑器（VS Code / Typora / Obsidian）写作
- **Git 版本管理** - 内容变更可追溯，误删可恢复，协作可合并
- **实时预览** - 本地开发时保存即刷新

### 🎨 展示效果

- **代码高亮** - 程序员友好的代码块展示
- **响应式布局** - 自动适配桌面端（移动端适配规划中）
- **主题系统** - 支持深色/浅色模式，可自定义主题

### ⚡ 性能表现

- **首屏快速** - 独立页面构建，首屏只加载必要资源
- **长期缓存** - 共享 chunk 自动提取，浏览器长期缓存
- **零数据库** - 内存索引，微秒级查询响应

### 🔒 数据安全

- **文件存储** - 内容即文件，备份只需 `tar czvf`
- **JWT 鉴权** - 管理后台独立鉴权，支持多用户（预留）
- **乐观锁** - 防止并发编辑覆盖（预留）

## 快速开始

### 1. 克隆项目

```bash
git clone https://github.com/yourname/next-ai-ventus.git
cd next-ai-ventus
```

### 2. 启动后端

```bash
cd server
go mod download
go run cmd/server/main.go
# 服务运行在 http://localhost:8080
```

### 3. 启动前端

```bash
cd frontend
pnpm install
pnpm dev
# 服务运行在 http://localhost:3000
```

### 4. 创建第一篇文章

```bash
mkdir -p content/posts/2024-06-hello-world
cat > content/posts/2024-06-hello-world/meta.json << 'EOF'
{
  "id": "2024-06-hello-world",
  "title": "Hello World",
  "slug": "hello-world",
  "date": "2024-06-15",
  "tags": ["hello"],
  "status": "published"
}
EOF

cat > content/posts/2024-06-hello-world/content.md << 'EOF'
# Hello World

这是我的第一篇文章。
EOF
```

访问 http://localhost:3000 即可看到文章。

## 为什么叫 ventus

> "Ventus" 是拉丁语中的 "风"，寓意轻盈、自由、无处不在。

我们希望这个博客系统像风一样：
- **轻盈** - 不占用过多服务器资源
- **自由** - 数据完全由你掌控
- **无处不在** - 只要有文件系统，就能运行
