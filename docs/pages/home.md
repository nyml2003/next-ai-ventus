# 首页设计

## 页面目标

让读者快速了解博客内容，发现感兴趣的文章。

## 用户场景

| 场景 | 用户行为 |
|------|---------|
| 首次访问 | 通过 Hero 区域了解博主，浏览最新文章 |
| 老读者回访 | 查看最新文章，点击标签筛选感兴趣的内容 |
| 寻找特定文章 | 使用搜索或直接翻页查找 |

## 模块清单

### 1. Header（顶部导航）

**功能：**
- 展示博客 Logo/名称
- 导航链接：首页、关于、搜索
- 主题切换按钮（浅色/深色）
- 登录入口（点击跳转到管理后台）

**交互：**
- Logo 点击返回首页
- 主题切换即时生效，保存到 localStorage

### 2. Hero（首屏横幅，可选）

**功能：**
- 展示博客标题和一句话简介
- 营造个人品牌氛围

**示例：**
```
┌─────────────────────────────────────────┐
│                                         │
│         张三的技术博客                   │
│    分享 Web 开发、架构设计与生活         │
│                                         │
└─────────────────────────────────────────┘
```

**说明：**
- 可在后台关闭，关闭后首屏直接展示文章列表
- 适合个人品牌较强的博主

### 3. PostList（文章列表）

**功能：**
- 展示文章卡片列表
- 每篇文章显示：标题、摘要、日期、标签

**文章卡片结构：**
```
┌─────────────────────────────────────────┐
│  如何设计一个高性能的博客系统              │
│  发表于 2024-06-15 · 架构, Go           │
│                                         │
│  本文介绍了一个无数据库的博客系统架构      │
│  设计思路，包括 BFF、文件存储等...        │
└─────────────────────────────────────────┘
```

**交互：**
- 点击卡片进入文章详情页
- 点击标签筛选该标签下的文章
- 滚动到底部自动加载更多（或分页）

### 4. Sidebar（侧边栏，可选）

**功能：**
- 博主简介（头像、一句话介绍）
- 热门文章 Top 5
- 标签云

**布局：**
- 桌面端：右侧固定宽度（300px）
- 平板/移动端：隐藏或折叠到底部

### 5. Footer（页脚）

**功能：**
- 版权信息
- 备案号（如需要）
- 技术栈声明（Powered by ventus）

## 空状态

**场景：博客没有任何文章**

```
┌─────────────────────────────────────────┐
│              Header                     │
├─────────────────────────────────────────┤
│                                         │
│            📝 欢迎来到 ventus           │
│                                         │
│    还没有文章，登录后台创建第一篇吧        │
│                                         │
│         [前往管理后台]                   │
│                                         │
└─────────────────────────────────────────┘
```

## 筛选状态

**场景：用户点击标签筛选**

- URL 变为 `/?tag=架构`
- PostList 标题变为「标签：架构」
- 显示「清除筛选」按钮，点击返回全部文章
