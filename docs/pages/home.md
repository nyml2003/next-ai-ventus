# 首页设计

## 页面目标

让读者快速了解博客内容，发现感兴趣的文章。

## 用户场景

| 场景 | 用户行为 |
|------|---------|
| 首次访问 | 浏览最新文章列表 |
| 老读者回访 | 查看最新文章，点击标签筛选感兴趣的内容 |
| 寻找特定文章 | 直接翻页查找 |

## 模块清单

### 1. Header（顶部导航）

**BFF 模块：`header`**（首屏模块）

**功能：**
- 展示博客 Logo/名称
- 导航链接：首页、关于、搜索
- 主题切换按钮（浅色/深色）
- 登录入口（点击跳转到管理后台）

**交互：**
- Logo 点击返回首页
- 主题切换即时生效，保存到 localStorage

### 2. Hero（首屏横幅，可选）【P1】

> **MVP 版本不做**，直接进入文章列表

**BFF 模块：`hero`**（首屏模块）

**功能：**
- 展示博客标题和一句话简介
- 营造个人品牌氛围

**P1 版本实现：**
- 可在后台开启/关闭
- 关闭后首屏直接展示文章列表

### 3. PostList（文章列表）

**BFF 模块：`postList`**（首屏核心模块）

**功能：**
- 展示文章卡片列表
- 每篇文章显示：标题、摘要、日期、标签

**文章卡片结构：**
```
┌─────────────────────────────────────────┐
│  如何设计一个高性能的博客系统              │
│  发表于 2024-06-15 · 架构, Go           │
│                                         │
│  本文介绍了一个无数据库的博客系统架构      │
│  设计思路，包括 BFF、文件存储等...        │
└─────────────────────────────────────────┘
```

**交互：**
- 点击卡片进入文章详情页
- 点击标签筛选该标签下的文章
- 滚动到底部自动加载更多（或分页）

### 4. Sidebar（侧边栏，可选）【P1】

> **MVP 版本不做**，标签筛选通过 URL 参数 `?tag=xxx` 实现

**BFF 模块：`sidebar`**（首屏模块）

**功能（P1）：**
- 博主简介（头像、一句话介绍）
- 热门文章 Top 5
- 标签云

**布局：**
- 桌面端：右侧固定宽度（300px）

**MVP 替代方案：**
- 标签筛选：文章卡片上的标签可点击，跳转到 `/?tag=xxx`
- 博主信息：暂时不展示，或放入 Footer

### 5. Footer（页脚）

**BFF 模块：`footer`**（可选）

**功能：**
- 版权信息
- 备案号（如需要）
- 技术栈声明（Powered by ventus）

## 筛选状态

**场景：用户点击标签筛选**

- URL 变为 `/?tag=架构`
- PostList 标题变为「标签：架构」
- 显示「清除筛选」按钮，点击返回全部文章
